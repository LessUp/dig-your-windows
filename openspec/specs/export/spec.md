# Export Specification

> **Domain**: export
> **Version**: 1.0.0
> **Status**: accepted
> **Last Updated**: 2026-04-17

## Overview

本文档定义 DigYourWindows 的报告导出 API。工具可以生成 JSON 和 HTML 格式的诊断报告。

## Export Formats

### 1. JSON Export

主要导出格式，结构化的 JSON 文档包含所有诊断数据。

#### 接口: ReportService.ExportToJson()

**方法签名**：

```csharp
public Task<string> ExportToJson(DiagnosticData data)
```

**输入**：
- `data`: 包含完整诊断信息的 DiagnosticData 对象

**输出**：
- `string`: JSON 格式的诊断报告

**示例输出**：

```json
{
  "version": "1.0.0",
  "exportDate": "2026-04-17T10:30:00Z",
  "toolVersion": "1.1.0",
  "data": {
    "timestamp": "2026-04-17T10:30:00Z",
    "systemInfo": { ... },
    "hardwareData": { ... }
  }
}
```

**元数据字段**：

| 字段 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `version` | string | 是 | 导出格式版本 |
| `exportDate` | string (ISO 8601) | 是 | 导出时间戳 |
| `toolVersion` | string | 是 | 生成报告的工具版本 |
| `data` | object | 是 | 完整诊断数据 |

### 2. HTML Export

生成自包含的 HTML 报告，用于离线查看。

#### 接口: ReportService.ExportToHtml()

**方法签名**：

```csharp
public Task<string> ExportToHtml(DiagnosticData data)
```

**输入**：
- `data`: DiagnosticData 对象

**输出**：
- `string`: 完整的 HTML 文档

**HTML 结构**：

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>DigYourWindows Report - [Date]</title>
    <style>/* 内嵌 CSS */</style>
</head>
<body>
    <header>
        <h1>System Diagnostic Report</h1>
        <p>Generated: [Date]</p>
    </header>
    <main>
        <section id="system-info">...</section>
        <section id="hardware-info">...</section>
        <section id="event-logs">...</section>
        <section id="performance">...</section>
    </main>
    <footer>
        <p>Generated by DigYourWindows v1.1.0</p>
    </footer>
</body>
</html>
```

## Error Handling

### 异常类型

所有导出操作可能抛出以下异常：

#### ReportException.InvalidData

当诊断数据无效或缺少必需字段时抛出。

```csharp
throw ReportException.InvalidData("SystemInfo is null");
```

**属性**：
- `ErrorType`: "InvalidData"
- `MissingField`: 缺失的字段名

#### ReportException.FileAccessError

当文件系统访问失败时抛出。

```csharp
throw ReportException.FileAccessError("Cannot write to path: C:\\output\\report.json");
```

**属性**：
- `ErrorType`: "FileAccessError"
- `Path`: 尝试的文件路径

## Data Validation

导出前执行以下验证：

1. **必需字段**：DiagnosticData 中所有必需字段必须存在
2. **数据类型**：所有值必须匹配预期类型
3. **日期格式**：所有日期必须是有效的 ISO 8601 格式
4. **数值范围**：所有数值必须在有效范围内

## File Naming Convention

导出文件遵循以下命名模式：

```
DigYourWindows_Report_[YYYY-MM-DD]_[HH-mm-ss].[json|html]
```

**示例**：
- `DigYourWindows_Report_2026-04-17_10-30-00.json`
- `DigYourWindows_Report_2026-04-17_10-30-00.html`

## Security Considerations

1. **路径清洗**：验证文件路径防止目录遍历攻击
2. **数据脱敏**：敏感信息（用户名、序列号）被脱敏
3. **无外部请求**：导出完全本地，无网络请求
4. **文件权限**：创建的文件使用限制性权限

## Performance Requirements

| 操作 | 目标性能 | 备注 |
|------|---------|------|
| JSON 导出 | < 500ms | 典型诊断数据 |
| HTML 导出 | < 1000ms | 包括模板渲染 |
| 文件写入 | < 200ms | 取决于存储介质 |
